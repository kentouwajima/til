# Linuxパーミッション（権限）速習チートシート

## 1. パーミッションの基本構造
Linuxの権限は「**誰に**」「**何を**」許可するかで決まります。
`ls -l` で表示される `rwxrwxrwx` という9文字は、3文字ずつの3グループに分かれています。

| 順番 | 対象 | 英語 (Who) | 意味 |
| :--- | :--- | :--- | :--- |
| **左の3文字** | **所有者** | Owner (User) | ファイルを作った本人（自分） |
| **中の3文字** | **グループ** | Group | 同じチームや部署のメンバー |
| **右の3文字** | **その他** | Others | 上記以外の全員（Web訪問者など） |

---

## 2. 文字と数字の変換ルール（足し算）
記号（r, w, x）にはそれぞれ「点数」が決まっており、その**合計点**で権限を表します。

| 記号 | 意味 | 英語 | 点数 | できること |
| :---: | :--- | :--- | :---: | :--- |
| **r** | 読み取り | **R**ead | **4** | ファイルの中身を見る / フォルダ内のファイル一覧を見る |
| **w** | 書き込み | **W**rite | **2** | 編集・保存・削除 / フォルダ内にファイルを作る |
| **x** | 実行 | e**X**ecute | **1** | プログラムを動かす / **フォルダの中に入る(cd)** |
| **-** | 権限なし | None | **0** | 何もできない |

### 計算の例
3文字の点数を足し算して、1つの数字にします。

* **`rwx`** (全権限) = 4 + 2 + 1 = **7**
* **`rw-`** (読み書き) = 4 + 2 + 0 = **6**
* **`r-x`** (読み実行) = 4 + 0 + 1 = **5**
* **`r--`** (読むだけ) = 4 + 0 + 0 = **4**

---

## 3. よく使うパーミッション設定例（Webサーバー編）
3つのグループ（自分・グループ・他人）の数字を並べて指定します。

### 📂 755 (ディレクトリの基本)
* **構成:** `rwx` (7) / `r-x` (5) / `r-x` (5)
* **意味:** 自分は自由。他人は「見る・入る」だけ。
* **用途:** Web公開用のフォルダ（`html`, `www` など）。

### 📄 644 (ファイルの基本)
* **構成:** `rw-` (6) / `r--` (4) / `r--` (4)
* **意味:** 自分は編集OK。他人は「見る」だけ。
* **用途:** HTMLファイル、画像、PHPファイルなど。

### 🔒 600 (自分専用)
* **構成:** `rw-` (6) / `---` (0) / `---` (0)
* **意味:** 自分だけが読み書きできる。他人は一切触れない。
* **用途:** SSHの秘密鍵、パスワードが書かれた設定ファイル。

### ⚠️ 777 (全開放・危険！)
* **構成:** `rwx` (7) / `rwx` (7) / `rwx` (7)
* **意味:** 世界中の誰でも、読み・書き・**削除**ができる。
* **用途:** 基本的に使用禁止（一時的なテストでのみ使用し、すぐ戻すこと）。

---

## 4. コマンドの使い方

### chmod (Change Mode)
権限を変更するコマンドです。

```bash
# ファイルの権限を変える（例：自分は書き込めるが、他人は見るだけの状態にする）
chmod 644 index.html

# ディレクトリの権限を変える
chmod 755 wp-content

# ディレクトリの中身も含めて一括で変える（-R オプション）
# ※WordPressなどでよく使う
chmod -R 755 wp-content
